AI Assistant

Duration: 6 months
Role: Lead Developer
Status: Completed

Overview

A fully modular, offline-first AI-powered personal assistant built for Windows. Designed to process both voice and text commands, the system performs real-time automation, system control, and intelligent task handling. The project supports wake-word detection, multi-intent processing, plugin-based extensions, and an integrated GUI for seamless interaction.

Key Features

  • Wake-Word Activation: Always-on listener responds to “Hey Assistant” instantly
  • Voice & Text Input: Dual I/O pipeline with Google Speech API fallback for high accuracy
  • Task Automation: Opens apps, performs calculations, controls volume, manages files
  • Smart Notes & Reminders: Stores notes, reminders, and command history locally
  • Weather & API Integrations: Real-time weather, dictionary, and knowledge lookups
  • Human-like Interaction: Rule-based NLP for natural, conversational responses
  • GUI Interface: Custom Tkinter interface with dark mode, animations, and status indicators
  • Plugin Architecture: Add or remove features without modifying core logic
  • Offline First: Uses Vosk STT + pyttsx3 TTS locally; online only when necessary

Technologies Used

Python 3.11 TensorFlow Tkinter (Custom GUI) Vosk Offline Speech Recognition Google SpeechRecognition API (Fallback for high accuracy) pyttsx3 Offline Text-to-Speech Custom NLP Engine (Intent Parser + Dispatcher) JSON-based Local Storage (notes, reminders, history)

Additional Libraries

PyAudio / sounddevice Requests (API integration) psutil (system monitoring) subprocess (automation) ctypes / pycaw (system volume control)

Architecture Highlights

Modular Folder Structure separating core logic, UI, plugins, tasks, utilities, and data

Event-driven workflow: wake-word → listen → parse intent → execute → respond

Extensible plugin system for new commands (music control, calendar, browser automation, etc.)

Offline priority ensures privacy and fast response time

Challenges & Solutions

Challenge: Ensuring fast, accurate speech recognition in an offline-first system.

Solution: Implemented a dual-mode STT engine: Primary: Google Speech API for accuracy & speed
Secondary: Vosk offline model as a fallback Added noise filtering + wake-word activation to increase reliability
Result: Significantly improved response rate and accuracy, even with background noise.

Results & Impact

  • 50% faster task execution compared to manual desktop operations
  • Processes commands in under 0.5 seconds with Google mode
  • Fully open-source, modular, and ready for future extensions like GPT-based reasoning
