Hospital Management System

Duration: 2 months
Role: Lead Developer
Status: Completed

Overview

The Hospital Management System is a role-based web application designed to streamline medical appointment workflows between patients and doctors. The platform supports secure login for both user types, appointment scheduling, and real-time confirmation. It maintains structured medical records, automates scheduling logic, and reduces manual administrative effort.

Key Features

  • Doctor Module: Manage availability and schedules, View incoming appointment requests, Approve / reject patient bookings, Access patient records and visit history
  • Patient Module: Register and log in securely, Search doctors by specialization, Book, cancel, or reschedule appointments, View appointment history and status updates
  • System Features: Multi-role authentication and authorization, Real-time notification of booking status, Structured database for doctors, patients, and appointments, User-friendly and responsive UI

Technologies Used

HTML CSS JavaScript Java MySQL Apache

Additional Libraries

JDBC

Architecture Highlights

Role-based access implemented with separate login dashboards

Modular MVC-style structure separating UI, business logic, and persistence

MySQL relational schema storing:Doctors and specializations, Patient profiles, Appointment metadata (time, date, status)

Secure session handling for both doctor and patient logins

Validation layer to prevent overlapping appointment bookings

Challenges & Solutions

Challenge: Avoiding overlapping or double-booked appointment times.
Role-based separate access flows.
Maintaining secure user authentication.
Keeping the interface easy for non-technical users.

Solution: Implemented availability checks and appointment conflict logic.
Designed two dashboards with controlled feature access.
Used hashed passwords and session-based login protection.
Created clean, responsive UI with intuitive navigation.

Results & Impact

  • Reduced appointment processing time by replacing manual systems
  • Improved scheduling efficiency and availability visibility
  • Centralized patient and doctor information in a structured database
  • Demonstrated full-stack development skills with authentication, CRUD functionality, database design, and UI workflow implementation
  • OpSuitable for deployment in clinics, academic training, or small-scale hospital use
